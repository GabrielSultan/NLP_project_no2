\begin{abstract}
This report describes an end-to-end natural language processing project on French insurance customer reviews: data preparation and exploratory analysis, distributional and neural representations (Word2Vec, FastText, LDA), supervised thematic classification (TF-IDF with logistic regression and fine-tuned DistilBERT), and a Streamlit application combining summarization, sentiment analysis, retrieval-augmented question answering, and a multi-stage similar-review search pipeline.
\end{abstract}

\section{Dataset and objectives}
The corpus consists of French reviews with metadata such as insurer (\texttt{assureur}), rating (\texttt{note}), product type, and publication dates. The analytical goal is to surface recurring themes (claims, customer service, pricing, reimbursements, etc.), compare classical and transformer-based classifiers, and expose the results through an interactive tool suitable for qualitative exploration.

\section{``Cleaned'' data and time-saving scale variables}
The exported ``cleaned'' table comes from a pragmatic pipeline with \textbf{scale factors} on the most expensive steps: spell correction uses \texttt{SPELL\_SCALE = 0.01}, translation \texttt{TRANSLATE\_SCALE = 0.005}, and French abstractive summaries \texttt{RESUME\_FRAC = 0.02}. In a final production pass, these values would simply be set to \texttt{1.0}. The Streamlit preprocessing path (\texttt{review\_preprocess.py}) reuses the core normalization needed for thematic inference and does not depend on those partial corrections.

\section{Preprocessing for modeling}
Token-level preprocessing follows a fixed recipe: apostrophe normalization, punctuation removal, length and digit filtering, French lemmatization with \texttt{simplemma}, and stopword removal (notebook artifact when available, otherwise NLTK). The goal is to keep a compact vocabulary while preserving insurance-specific signal.

\textbf{Lemmatization vs.\ stemming.} Because French is morphologically rich, stemming often produces opaque roots and merges unrelated forms. Lemmatization keeps readable canonical forms, improves interpretability for LDA, TF-IDF, and Word2Vec, and reduces vocabulary fragmentation.

\section{Exploratory analysis and n-grams}
Unigrams, bigrams, and trigrams were compared before and after cleaning with \texttt{CountVectorizer} (\texttt{max\_features=50}) on a sample of 5{,}000 raw texts versus all processed texts. This cap keeps the plots readable; it is only for EDA and is unrelated to the larger vocabularies used later for supervised learning.

\section{Word2Vec and FastText embeddings}
\textbf{Word2Vec} was trained with Gensim on lemmatized review tokens using \texttt{vector\_size=200}, \texttt{window=5}, \texttt{min\_count=7}, \texttt{workers=1}, and \texttt{seed=42} (these values were chosen through empirical trials). The embedding dimension and context window follow common defaults for medium-sized corpora, while \texttt{min\_count} suppresses rare tokens that would otherwise add noise. See technical reference~\cite{ref:gensim-w2v}. Two-dimensional PCA projections, in which words are colored by average rating, help interpret semantic neighborhoods (Figure~\ref{fig:w2vft}).

\textbf{FastText} French crawl vectors (\texttt{cc.fr.300}) were loaded without additional training to contrast a \textbf{pretrained} subword model with the domain-trained Word2Vec space~\cite{ref:fasttext-vec}.FastText handles morphological variants and OOV pieces via character n-grams, which often yields different nearest neighbors than a corpus-specific Word2Vec model. In the figures below, the separation by average rating is weaker than for Word2Vec.

\section{LDA topic exploration}
Latent Dirichlet Allocation used bigram documents, a dictionary/TF-IDF corpus, and \texttt{LdaMulticore} with \texttt{alpha='asymmetric'} and \texttt{eta='auto'}. The number of topics $k$ was selected from $\{4,6,8,10,12\}$ by maximizing the $c_v$ coherence score from Gensim's \texttt{CoherenceModel}, which balances interpretability against granularity. TF-IDF weighting reduces the dominance of ultra-frequent terms before topic induction.

\textbf{From unsupervised themes to supervised targets.} LDA is valuable for exploratory structure: it surfaces co-occurrence patterns without predefined labels and helps sanity-check whether the corpus clusters along plausible semantic directions. For this project's goals, however, that strength is also a limit. Topics are latent mixtures, their names and boundaries remain partly subjective, and there is no single canonical mapping from a topic vector to a fixed menu of insurance-facing categories that could drive filtering, reporting, or evaluation. To attach each review to a discrete, auditable \texttt{thematique} and to train classifiers whose accuracy can be measured, the analysis therefore moves to \emph{supervision}: first by constructing reproducible weak labels aligned with business axes (next section), then by learning models that generalize beyond exact keyword overlap (Section~\ref{sec:supervised-thematic}).

\section{Deterministic thematic labels for supervision}
\label{sec:deterministic-labels}
Supervised models in the notebook (Section~7) need a target column \texttt{thematique}. The project therefore uses a \textbf{deterministic weak-labelling} procedure: reproducible, inexpensive, and aligned with insurance-facing axes.

\textbf{Rationale.} Keyword rules are easy to audit, and a small Word2Vec neighbourhood expansion captures close variants without hand-curating every form. The trade-off is simple: one label per review, fixed priority when several rules match, and some label noise.

\textbf{Construction (notebook).} A dictionary \texttt{THEMATIQUES} maps each theme to truncated keyword fragments consistent with \texttt{avis\_traite}. The first two seeds of each theme are optionally expanded with up to three Word2Vec neighbours each, then \texttt{detect\_thematique} scans themes in fixed order and assigns the first matching label; otherwise it returns \texttt{autre}.

\section{Supervised thematic classification}
\label{sec:supervised-thematic}
This section compares \textbf{two alternative supervised approaches} trained to predict the same \texttt{thematique} column built in Section~\ref{sec:deterministic-labels}. We tested a \textbf{classical sparse-text model} and a \textbf{transformer model} independently, then compares them on the same stratified 80/20 split to decide which one is most suitable for the Streamlit application.

\textbf{Approach 1: TF-IDF + logistic regression.} The first solution is a standard bag-of-words baseline: \texttt{TfidfVectorizer(max\_features=5000, ngram\_range=(1,2))} converts the preprocessed reviews into sparse unigram/bigram features, and \texttt{LogisticRegression(max\_iter=500, random\_state=42)}~\cite{ref:sklearn-tfidf} learns a linear separator between thematic classes. This approach is simple, fast, and interpretable, so it is a useful reference point for judging whether a heavier neural model is really necessary.

\textbf{Approach 2: fine-tuned DistilBERT.} We fine-tuned \texttt{distilbert-base-multilingual-cased}~\cite{ref:distilbert} end to end with the Hugging Face \texttt{Trainer} API~\cite{ref:hf-trainer}. The notebook uses \texttt{MAX\_LEN=128}, learning rate $2\times 10^{-5}$, batch sizes 16/32, two epochs, evaluation at each epoch, and the same stratified split as the classical baseline. The resulting weights are stored in \texttt{artifacts/distilbert\_thematic/} for later reuse in the application.

\textbf{Comparison on the held-out test set.} Both models are evaluated on the same 4{,}821-review test split. The TF-IDF baseline reaches about 90\% accuracy, macro F1 $\approx 0.69$, and weighted F1 $\approx 0.89$. The classification report shows why macro F1 is much lower than weighted F1: common classes such as \texttt{service\_client}, \texttt{prix}, or \texttt{sinistres} are predicted reasonably well, but rarer or less distinct labels suffer much more.

Fine-tuned DistilBERT performs much better on that same split: accuracy rises to about 97\%, macro F1 to about 0.95, and weighted F1 to about 0.97. Figure~\ref{fig:model-comparison} summarizes the gap between the two approaches. The gain is not only global: it is also visible on minority or semantically close classes, which is exactly where the bag-of-words approach loses the most information. The confusion matrix in Figure~\ref{fig:distilbert-cm} is strongly diagonal.

\textbf The best model is not simply the fastest one, but the one that generalizes most reliably beyond explicit keyword overlap. The comparison shows that DistilBERT is clearly superior to the classical ML baseline on the shared test set. It also remains lighter than full BERT, so inference is still practical in an application that already runs additional models for summarization and sentiment analysis. For that reason, the Streamlit app keeps the fine-tuned DistilBERT classifier rather than the TF-IDF + logistic regression alternative.

\section{Streamlit application architecture}
Figure~\ref{fig:arch} sketches the main flows. The \textbf{single-review} tab runs preprocessing, DistilBERT thematic probabilities (metrics and rationale in \S\ref{sec:supervised-thematic}), a French summarizer, a multilingual sentiment head, and similar-review retrieval. The \textbf{RAG} tab embeds chunked reviews and calls a local LLM for grounded answers.

\textbf{Summarization:} uses \texttt{moussaKam/barthez-orangesum-abstract}~\cite{ref:barthez} with deterministic generation and length bounds derived from review length.

\textbf{Sentiment:} uses \texttt{nlptown/bert-base-multilingual-uncased-sentiment}~\cite{ref:sentiment-bert}; five-star outputs are mapped to coarse polarity.

\textbf{Similar reviews:} The retrieval stack reuses the Project~1 approach: BM25~\cite{ref:bm25} over tokenized reviews (cap 4{,}000 rows for latency), then semantic reranking with \path{sentence-transformers/all-MiniLM-L6-v2}~\cite{ref:minilm-l6}, and optional Ollama reranking with \texttt{llama3.2}~\cite{ref:ollama-llama}. The final interface returns five matches.

\textbf{RAG:} Reviews are chunked into 90-word windows with 20-word overlap, embedded with \path{paraphrase-multilingual-MiniLM-L12-v2}~\cite{ref:paraphrase-l12}, retrieved by cosine similarity, and passed to Ollama \texttt{llama3.2}~\cite{ref:ollama-llama} for grounded answers.

\textbf{Demonstration.} This written overview cannot capture every interaction detail of the Streamlit
interface (tab layout, index build button, insurer filter, error states when Ollama is offline, etc). A separate explanatory video shows the interface end to end, including the RAG tab and the single-review workflow.

\section{Limitations and conclusion}
The pipeline mixes research prototypes (partial spell-check and translation coverage, capped similar-review corpus, RAG document limits) with core modeling (stratified splits, coherence-driven LDA, DistilBERT fine-tuning). Production use would require full-data cleaning, systematic evaluation on held-out temporal splits, monitoring of Ollama latency and failures, and privacy review of third-party model endpoints if any were introduced. Overall, the project demonstrates a coherent stack from classical NLP through transformers to retrieval-augmented generation.
